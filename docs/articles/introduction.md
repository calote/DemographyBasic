``` r

install.packages("remotes")
```

## Use

``` r

suppressWarnings(suppressMessages(library(DemographyBasic)))
```

``` r

suggests <- c("ggthemes", "LexisPlotR", "mapSpain", "tidyr")
for (pkg in suggests) {
  if (!requireNamespace(pkg, quietly = TRUE)) {
    msg <- switch(
      tolower(Sys.info()["sysname"]),
      linux = paste0(
        "Package '", pkg, "' is required.\n",
        "On Ubuntu/Debian, first install system libraries:\n",
        "  sudo apt-get install -y libudunits2-dev libproj-dev libgdal-dev libgeos-dev\n",
        "Then run: install.packages('", pkg, "')"
      ),
      darwin = paste0(
        "Package '", pkg, "' is required.\n",
        "On macOS, first install system libraries with Homebrew:\n",
        "  brew install pkg-config gdal proj geos udunits\n",
        "Then run: install.packages('", pkg, "')"
      ),
      windows = paste0(
        "Package '", pkg, "' is required.\n",
        "Install it with: install.packages('", pkg, "')"
      )
    )
    stop(msg)
  }
  library(pkg, character.only = TRUE)
}
```

## Crear gráficos con ggplot2

En este apartado haremos una introducción muy básica a las capacidades
gráficas del paquete “**ggplot2**” que forma parte del **sistema
“tidyverse”** y es posiblemente el paquete R de **tratamiento gráfico
más usado en la actualidad**.

La idea que se usa en este paquete es añadir paso a paso las distintas
capas que constituirán el gráfico final:

1.  Especificar los datos (tibble o data.frame).
2.  Indicar las columnas que se van a utilizar y el papel que
    desempeñarán en el gráfico
    ([`aes()`](https://ggplot2.tidyverse.org/reference/aes.html)).
3.  Añadir el tipo o tipos de gráficos que se van a representar.
4.  Especificar las características particulares del gráfico, en
    particular las leyendas o texto explicativo que llevará.

Veremos a través de ejemplos cómo se construyen gráficos con el paquete
“ggplot2”, en concreto construiremos los siguientes tipos de gráficos:

- Diagramas de barras o columnas
- Diagramas de líneas

- `datos2_mej.RData`

Los pasos para la **construcción con “ggplot2”** del diagrama de barras

    [`labs()`](https://ggplot2.tidyverse.org/reference/labs.html), en la

#### Ordenar barras por orden descendente de valor

**Ejemplo**. Para presentar las columnas siguiendo algún tipo de orden
(por defecto, las ordena según el orden alfabético) se puede utilizar la
función [`reorder()`](https://rdrr.io/r/stats/reorder.factor.html).

argumento indica la columna que se usará para las etiquetas, y la
segunda columna será para indicar el orden en el que aparecerán (si se
quiere presentar en orden contrario se debe colocar un signo “-” delante
del segundo argumento).

``` r
ggplot(datos_CCAA,aes(x=reorder(CCAA,TOTALCCAA),y=TOTALCCAA)) +
```
